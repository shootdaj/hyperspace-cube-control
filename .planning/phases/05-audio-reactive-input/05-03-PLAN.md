# Plan 05-03: FFT Analysis & AudioPlugin InputPlugin

## Objective
Implement the AudioPlugin that wraps the AudioContextManager and implements the InputPlugin interface. The plugin's `tick()` method reads FFT data from the AnalyserNode and returns it as `FrameData { type: 'audio', spectrum }`.

## Context
- @src/core/pipeline/types.ts — InputPlugin, FrameData interfaces
- @src/plugins/inputs/ManualPaintPlugin.ts — Reference InputPlugin implementation
- @src/plugins/inputs/AudioContextManager.ts — Audio lifecycle manager (from 05-02)
- @src/stores/audioStore.ts — Audio state store (from 05-01)

    Create `src/plugins/inputs/AudioPlugin.ts`:
    - Class AudioPlugin implements InputPlugin
    - Internal AudioContextManager instance (created in constructor or lazily)
    - `frequencyData: Uint8Array` — reusable buffer, size = 1024 (fftSize 2048 / 2)
    - `spectrumData: Float32Array` — reusable buffer for normalized output
    - `initialize(ctx: PluginContext)`: store pluginContext, allocate buffers
    - `tick(deltaMs: number)`:
      1. Get analyser from AudioContextManager
      2. If null, return null
      3. Call analyser.getByteFrequencyData(frequencyData)
      4. Normalize to Float32Array: spectrumData[i] = frequencyData[i] / 255
      5. Return { type: 'audio', spectrum: spectrumData }
    - `async startAudio(deviceId?: string)`: delegate to manager.start(deviceId)
    - `stopAudio()`: delegate to manager.stop()
    - `setGain(v: number)`: delegate to manager.setGain(v), update audioStore

    - `destroy()`: manager.destroy()
