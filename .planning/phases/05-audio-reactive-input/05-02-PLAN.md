# Plan 05-02: AudioContext Initialization & Lifecycle

## Objective
Create the AudioContext manager that lazily creates and resumes the AudioContext only on user gesture. Handles suspended state monitoring, stream acquisition for a selected device, and the GainNode → AnalyserNode audio graph.

## Context
- @src/stores/audioStore.ts — Audio state store (from 05-01)
- @src/plugins/inputs/audioDeviceService.ts — Device enumeration (from 05-01)
- @src/core/pipeline/types.ts — Plugin interfaces
- @.planning/phases/05-audio-reactive-input/05-RESEARCH.md — Research

  <behavior>
    - constructor does NOT create AudioContext (lazy)
    - start(deviceId) creates AudioContext if not exists, resumes if suspended
    - start(deviceId) calls getUserMedia with exact deviceId constraint
    - start(deviceId) builds graph: MediaStreamSource → GainNode → AnalyserNode
    - start(deviceId) updates audioStore state (isAudioActive=true, audioContextState)
    - stop() stops all stream tracks, disconnects nodes, sets isAudioActive=false
    - destroy() closes AudioContext, nullifies all references
    - setGain(value) updates GainNode.gain.value
    - getAnalyser() returns the AnalyserNode (or null if not started)
    - getState() returns audioContext.state (or null)
    - AnalyserNode configured: fftSize=2048, smoothingTimeConstant=0.8
    - start() with no existing context creates one with default sample rate
    - start() when already running with different device stops old stream and starts new one
  </behavior>
  <action>
    Create `src/plugins/inputs/AudioContextManager.ts`:
    - Class with private fields: audioContext, gainNode, analyserNode, mediaStream, sourceNode
    - `async start(deviceId?: string)`:
      1. If audioContext is null, create new AudioContext()
      2. If audioContext.state === 'suspended', await audioContext.resume()
      3. If existing mediaStream, stop all tracks and disconnect sourceNode
      4. Get stream via getUserMedia({ audio: deviceId ? { deviceId: { exact: deviceId } } : true })
      5. Create MediaStreamAudioSourceNode from stream
      6. Create GainNode, set gain to audioStore.getState().gain
      7. Create AnalyserNode, fftSize=2048, smoothingTimeConstant=0.8
      8. Connect: source → gain → analyser (do NOT connect to destination — analysis only)
      9. Update audioStore: isAudioActive=true, audioContextState=audioContext.state
      10. Listen to audioContext.onstatechange → update audioStore.audioContextState
    - `stop()`: Stop stream tracks, disconnect nodes, set audioStore.isAudioActive=false
    - `destroy()`: stop() then audioContext.close()
    - `setGain(v: number)`: gainNode.gain.setValueAtTime(v, audioContext.currentTime)
    - `getAnalyser()`: returns analyserNode or null
    - `getState()`: returns audioContext?.state ?? null

    Mock AudioContext, AnalyserNode, GainNode, MediaStreamAudioSourceNode, getUserMedia in tests.
    Use a factory pattern for AudioContext creation so tests can inject mocks.
  </action>
  <verify>
    <automated>npx vitest run src/plugins/inputs/__tests__/AudioContextManager.test.ts</automated>
  </verify>

    - Audio graph is correctly wired: source → gain → analyser

    - All tests pass
  </done>
