  <files>src/App.tsx, src/visualization/CubeMesh.tsx, src/control/PaintControls.tsx</files>
  <action>
    1. **App.tsx** — Initialize ManualPaintPlugin and WLEDPaintOutput:
       - Create a module-level or useRef instances:
         ```typescript
         const paintPlugin = new ManualPaintPlugin();
         const paintOutput = new WLEDPaintOutput();
         ```
       - On mount (or when connection established), initialize the paint plugin
       - Export/expose these via a simple module-level getter so CubeMesh and PaintControls can access them
       - OR use a small Zustand store / module singleton pattern
       - Register ManualPaintPlugin with PluginRegistry for discoverability

    2. **CubeMesh.tsx** — Update paintLed to also send to WLED:
       - After writing to ManualPaintPlugin and ledStateProxy, call:
         ```typescript
         paintOutput.sendPaint(plugin.getBuffer());
         ```
       - This triggers diff-based WLED send (throttled to 30fps)

    3. **PaintControls.tsx** — Wire clear/fill to WLED:
       - Clear button: calls plugin.fill(0,0,0), copies to ledStateProxy, calls paintOutput.sendAll(plugin.getBuffer())
       - Fill button: calls plugin.fill(r,g,b), copies to ledStateProxy, calls paintOutput.sendAll(plugin.getBuffer())
       - Use the same singleton paint plugin/output instances

    Singleton access pattern (simplest):
    ```typescript
    // src/plugins/inputs/paintSingleton.ts
    import { ManualPaintPlugin } from './ManualPaintPlugin';
    import { WLEDPaintOutput } from '../outputs/WLEDPaintOutput';

    export const paintPlugin = new ManualPaintPlugin();
    export const paintOutput = new WLEDPaintOutput();
    ```
    Import this in CubeMesh and PaintControls.
  </action>
