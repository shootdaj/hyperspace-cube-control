<objective>
Wire raycasting pointer events on the CubeMesh InstancedMesh to detect LED clicks/drags and route them through ManualPaintPlugin.

Purpose: This is the interaction layer that connects user gestures (click, drag) to the paint buffer. Users see instant visual feedback because paint writes to both the plugin buffer and ledStateProxy.

Output: Interactive painting on the 3D cube with single-click and drag support.
</objective>

  <name>Task 1: Paint store and CubeMesh pointer events</name>
  <files>src/stores/paintStore.ts, src/visualization/CubeMesh.tsx</files>
  <action>
    Create `src/stores/paintStore.ts` using Zustand:
    ```typescript
    interface PaintState {
      isPaintMode: boolean;
      brushSize: 'single' | 'edge' | 'face';
      color: [number, number, number]; // RGB 0-255
      setIsPaintMode: (v: boolean) => void;
      setBrushSize: (v: 'single' | 'edge' | 'face') => void;
      setColor: (rgb: [number, number, number]) => void;
    }
    ```
    Default: isPaintMode=false, brushSize='single', color=[255,255,255].

    Modify CubeMesh.tsx:
    1. Add a module-level singleton ref for ManualPaintPlugin instance (NOT React state):
       ```typescript
       let paintPluginInstance: ManualPaintPlugin | null = null;
       export function setPaintPlugin(plugin: ManualPaintPlugin | null) { paintPluginInstance = plugin; }
       export function getPaintPlugin(): ManualPaintPlugin | null { return paintPluginInstance; }
       ```
    2. Add refs inside CubeMesh: `paintingRef = useRef(false)`, `lastPaintedRef = useRef(-1)`
    3. Add pointer event handlers on the `<instancedMesh>`:
       - `onPointerDown`: if paintStore.isPaintMode, set paintingRef=true, call paintLed(e.instanceId), call e.stopPropagation()
       - `onPointerMove`: if paintingRef.current && e.instanceId !== lastPaintedRef.current, call paintLed(e.instanceId)
       - `onPointerUp`: set paintingRef=false, lastPaintedRef=-1
       - `onPointerLeave`: set paintingRef=false, lastPaintedRef=-1
    4. `paintLed(index)` function:
       - Reads color and brushSize from paintStore.getState() (not hook — avoid re-renders)
       - Based on brushSize:
         - 'single': plugin.setPixel(index, r, g, b)
         - 'edge': plugin.setEdge(getEdgeIndex(index), r, g, b)
         - 'face': get edge, get first face from getEdgeFaces(edge), plugin.setFaceEdges(face, r, g, b)
       - After writing to plugin, also directly update ledStateProxy.colors for instant visual:
         Copy relevant bytes from plugin.getBuffer() to ledStateProxy.colors
       - Set lastPaintedRef.current = index
    5. The cursor style should change to 'crosshair' when isPaintMode is true. Use Zustand subscription to set cursor on the canvas parent.
  </action>
  <verify>
    <automated>npx vitest run --dir src</automated>
  </verify>
  <done>CubeMesh responds to pointer events. Click on LED identifies correct instance. Drag paints multiple LEDs. paintStore holds brush state.</done>
