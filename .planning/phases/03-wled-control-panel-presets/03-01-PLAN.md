# Plan 03-01: WLED State Round-Trip

## Objective
Wire the WLED state round-trip so that outgoing control changes flow through cubeStateStore → WLEDRestClient, and incoming WebSocket state updates reconcile the store. Components never call WLED directly. This is the foundation all other Phase 3 plans build on.

## Context
- @src/core/store/cubeStateStore.ts — has actions like setOn, setBrightness, syncFromWLED
- @src/core/wled/WLEDRestClient.ts — serialized queue for REST calls
- @src/core/wled/WLEDWebSocketService.ts — singleton WebSocket
- @src/core/store/types.ts — WLEDState, CubeSegment types
- @src/core/wled/types.ts — WLEDMessage, WLEDState (WLED-side types)
