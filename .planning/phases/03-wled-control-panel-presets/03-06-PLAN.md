# Plan 03-06: Preset System

## Objective
Build a preset save/load/delete system that persists to localStorage. Users can save the current cube state as a named preset, reload it with one tap, and manage up to 50 presets.

## Context
- @src/core/store/cubeStateStore.ts — current state to save
- @src/core/wled/WLEDControlService.ts — to apply preset state
- @src/components/ui/dialog.tsx — for save dialog

  <name>Task 1: Preset store with localStorage persistence</name>
  <files>

  <behavior>
    - Test: savePreset creates a preset with name, id, timestamp, and current cube state
    - Test: savePreset stores to localStorage under 'hypercube-presets' key
    - Test: loadPresets reads from localStorage and populates store
    - Test: deletePreset removes preset by id and updates localStorage
    - Test: Max 50 presets enforced — saving 51st replaces oldest
    - Test: Preset survives simulated browser refresh (read from localStorage)
    - Test: applyPreset calls WLEDControlService with preset state values
  </behavior>
  <action>
    Create presetStore (Zustand):
    ```typescript
    interface Preset {
      id: string;
      name: string;
      createdAt: number;
      state: {
        on: boolean;
        bri: number;
        fx: number;
        pal: number;
        sx: number;
        ix: number;
        col: [number, number, number][];
      };
    }
    
    interface PresetStore {
      presets: Preset[];
      loadPresets: () => void;  // from localStorage
      savePreset: (name: string) => void;  // snapshot current cubeStateStore
      deletePreset: (id: string) => void;
      applyPreset: (id: string) => void;  // send to WLED via WLEDControlService
    }
    ```
    
    Implementation:
    - localStorage key: 'hypercube-presets'
    - savePreset: read cubeStateStore.getState(), create Preset, append, persist
    - If presets.length >= 50, remove oldest (by createdAt)
    - applyPreset: find preset, call WLEDControlService.batchUpdate with preset.state
    - Use crypto.randomUUID() for id generation
  </action>
  <verify>
    <automated>npx vitest run src/core/store/__tests__/presetStore.test.ts</automated>
  </verify>
  <done>Preset CRUD operations work with localStorage persistence and 50-preset limit</done>
</task>
